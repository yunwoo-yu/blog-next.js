저는 화면을 구현할 때 사용자가 보는 결과와 실제 처리 기준이 어긋나지 않는지를 중요하게 보는 프론트엔드 개발자입니다. 버튼을 누른 뒤 어떤 상태가 바뀌는지, 실패했을 때 무엇을 알려줘야 하는지, 운영자는 같은 데이터를 어떤 기준으로 처리해야 하는지까지 함께 확인하며 개발해 왔습니다.

슈터플랫폼실 프론트 엔지니어 역할도 이 관점과 맞닿아 있다고 생각합니다. 슈팅 게임의 웹·모바일 플랫폼은 단순한 부가 페이지가 아니라 플레이어가 게임을 계속 이해하고 즐기게 만드는 서비스입니다. 전적, 랭킹, 아이템, 이벤트, 거래, 공지, 백오피스처럼 게임 밖 화면에서 제공되는 정보가 정확해야 플레이어는 서비스를 신뢰하고, 운영자는 안정적으로 라이브 서비스를 이어갈 수 있습니다.

저는 사용자 서비스와 운영 어드민을 함께 개발해 온 경험이 있습니다. 커머스에서는 사용자가 확인하는 주문·결제 상태와 운영자가 처리하는 주문 상태가 같은 기준으로 움직이도록 화면과 API 처리 기준을 맞췄고, 금액이나 상태가 어긋나면 곧바로 운영 이슈가 되는 영역을 다뤘습니다. 검색 서비스에서는 여러 조건을 입력하는 화면을 공통화하고 한글 입력 오류를 해결했으며, 마이데이터 연동에서는 인증과 동의처럼 단계가 복잡한 기능을 사용자가 현재 상태를 이해할 수 있는 화면으로 정리했습니다.

살린에서는 AI 챗봇 서비스, 관리자 CMS, WebGL 기반 서비스, 공통 UI Kit을 개발하며 운영자가 콘텐츠와 설정을 직접 관리하는 구조를 만들었습니다. 사용자 화면만 잘 동작하는 것으로는 충분하지 않고, 운영 화면까지 함께 안정적이어야 서비스가 지속될 수 있다는 점을 이 과정에서 배웠습니다.

저는 실제 게임 사용자로서도 이 포지션에 관심이 있습니다. 게임을 오래 플레이하다 보면 인게임 플레이만큼 전적 확인, 이벤트 참여, 보상 확인, 아이템 관리 같은 외부 서비스의 완성도가 크게 느껴집니다. 불편한 화면은 금방 이탈로 이어지고, 정확한 정보와 빠른 반응은 게임에 대한 신뢰로 이어집니다.

제가 가진 강점은 이런 접점을 프론트엔드에서 구체적인 화면과 상태 처리로 옮기는 데 있습니다. 플레이어가 원하는 정보를 빠르게 확인하고, 운영자는 같은 기준으로 서비스를 관리할 수 있는 웹·모바일 플랫폼을 만드는 데 잘 맞는 개발자라고 생각합니다.
