Wello 내 지역 상품 커머스 서비스인 웰마켙의 사용자 화면과 운영 어드민을 함께 개발했습니다. 상품 조회, 장바구니, 주문서, 결제 완료, 주문내역으로 이어지는 화면을 구현하고, 사용자가 보는 금액과 실제 PG 결제 요청 금액이 같은 기준으로 계산되도록 프론트엔드 상태와 검증 로직을 설계했습니다.

NICE Payments 결제 연동 과정에서는 쿠폰, 포인트, 배송비가 반영된 최종 결제 금액을 화면과 요청 파라미터 양쪽에서 일관되게 다루도록 구현했습니다. 구매 이후에는 주문 상태 변화에 따라 사용자가 필요한 후속 행동을 이어갈 수 있도록 주문 상세와 운영 어드민의 주문 처리를 연결했습니다.

운영 어드민은 운영자와 입점업체가 같은 주문 기준을 보고 업무를 처리할 수 있도록 구성했습니다. 상품 운영, 주문 처리, 고객 응대, 프로모션 관리에 필요한 화면을 개발하고 권한에 따라 접근 가능한 기능이 달라지도록 API 처리를 정리했습니다. React, Next.js, TypeScript, TanStack Query, Zustand, Tailwind CSS, OpenAPI 기반으로 개발했으며 QA와 운영 이슈 대응까지 수행했습니다.

경력 기술 및 수행 직무와 역할
Wello-biz 상세검색은 사용자가 여러 키워드 조건을 조합해 지원사업과 조달사업을 찾는 핵심 기능이었습니다. 기존에는 같은 입력 UX가 여러 화면에 반복 구현되어 있었고, 한글 입력 중 Enter를 누르면 조합이 끝나지 않은 문자열이 검색 조건으로 추가되는 문제가 있었습니다.

이를 해결하기 위해 태그형 키워드 입력 컴포넌트를 공통화하고, IME 조합 상태와 일반 입력 확정을 분리해 한글 검색어가 안정적으로 추가되도록 개선했습니다. 동일한 컴포넌트를 11개 검색 입력 지점에 적용해 화면별로 달랐던 추가, 삭제, 초기화, 검증 동작을 하나의 기준으로 맞췄습니다.

React, Next.js, TypeScript, TanStack Query, Zod 기반으로 개발했으며, 반복 구현을 줄이면서 검색 입력의 안정성과 유지보수성을 함께 개선했습니다.

경력 기술 및 수행 직무와 역할
정책 추천 고도화를 위해 의료·통신·신용 마이데이터 연동이 필요했습니다. 단순 API 연동이 아니라 기관별 본인인증, 동의, 전송요구, 전자서명, 재동의, 철회, 전송 이력 확인까지 심사 기준에 맞춰 검증되어야 하는 규제성 플로우였습니다.

기관마다 단계 구성과 API 처리 방식이 달라 어댑터 패턴을 적용했습니다. 기관별로 필요한 단계, 요청 생성, 응답 정규화, 에러 매핑을 어댑터로 분리하고, 공통 로직은 단계 실행, 진행 상태 저장, 화면 전환, 실패 처리만 담당하게 했습니다. 진행 상태는 스텝 식별자는 URL 쿼리스트링에, 인증 중간값은 세션스토리지에 분리해 저장했고, 탭을 닫으면 민감한 중간 상태가 사라지도록 의도했습니다.

기관별 보완 요청이 들어와도 화면을 새로 만들지 않고 어댑터 단위 수정으로 대응할 수 있는 구조를 확보했습니다. 새 기관이 추가되더라도 어댑터만 추가하면 기존 화면과 공통 로직을 그대로 재사용할 수 있어, 규제 대응과 기관 확장을 분리해서 관리할 수 있게 됐습니다.

React, Next.js, TypeScript, TanStack Query, OpenAPI 기반으로 개발했으며, 인증, 동의, 전송요구, 철회로 이어지는 복잡한 절차를 사용자가 이해할 수 있는 단계형 화면으로 정리했습니다.

경력 기술 및 수행 직무와 역할
Wello는 기존 PC 화면에서도 모바일 중심 레이아웃을 그대로 사용하고 있어, 넓은 화면에서 정책과 지역 콘텐츠를 함께 탐색하기 어려웠습니다. PC 사용자가 홈에서 정책, 동네소식, 정책 활용 가이드로 자연스럽게 이어질 수 있도록 화면 구조를 다시 설계했습니다.

홈은 피드 중심 구조로 재배치하고, 통합검색은 검색 조건과 결과를 한 화면에서 빠르게 비교할 수 있도록 구성했습니다. 정책 상세는 핵심 요약과 신청에 필요한 정보를 섹션 단위로 정리해 사용자가 지원 가능 여부와 다음 행동을 빠르게 판단할 수 있도록 개선했습니다.

React, Next.js, TypeScript, TanStack Query, Tailwind CSS 기반으로 개발했으며, 개편 이후 GA4 기준 desktop 평균 참여 시간이 약 20% 증가했습니다.
